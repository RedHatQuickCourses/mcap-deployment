Assisted Clusters - Infrastructure Clusters

In this section, you will be deploying Infrastructure clusters using RHACM from Hub cluster. These Infrastructure clusters will be Single Node OpenShift (SNO) clusters.

Prerequisites

  1. sno1, sno2 and sno3 VMs are up and running.

  2. Download or copy your pull-secret and put it in pull_secret.txt file on hypervisor.

Deploy Infrastructure clusters as SNO clusters

  1. Login to web console of Hub cluster.

    Switch to the All Clusters from local-cluster.

  2. Create cluster using Create cluster.

  3. Select Host inventory.

  4. Select Standalone.

  5. Select Add new hosts.

  6. Provide cluster details.

    Cluster name: sno1

    Cluster set: default

    Base domain: lab.example.com

    Select latest available stable version of OpenShift from the menu

    Check box the Install single node OpenShift (SNO)

    1. Get the pull secret from pull_secret.txt.

    2. Provide the pull secret in Pull secret field.

      Click Next

  7. No automation template.

    Click Next

  8. Review and save.

    Click Save

  9. Add sno1 VM as host.

    Click Add hosts.

    1. Select Add hosts  With Discovery ISO.

    2. Here you need to provide the public ssh key of root user.

    3. Get the public ssh key of root user.

    4. Provide the public ssh key of root user in SSH public key field.

      Click Generate Discovery ISO to generate discovery ISO.

    5. Click Download Discovery ISO to download discovery ISO on hypervisor.

    6. This will open link in new tab.

      Click Advanced…​ to proceed.

      Click Accept the Risk and Continue to proceed.

    7. Move iso from Download directory to /var/lib/libvirt/images/ directory.

      Sample output
      [root@hypervisor ~]# mv Downloads/39c2b646-e122-435b-b85e-c8ec6dd18d04-discovery.iso /var/lib/libvirt/images/

    8. Boot the sno1 VM with discovery ISO.

      Shutdown the sno1 VM.

      Click Add hardware

      Select Device type  CDROM device and select Select or create custom image option.

      Click Manage.

      Select the discovery ISO and click Choose Volume.

      Select the discovery ISO and click Finish.

      Update the Boot device order to boot system with discovery ISO.

      Click Apply.

      Boot the sno1 VM and ensure it is booted with RHEL CoreOS (Live).

    9. In hub console, notice sno1 VM as host is discovered and select Approve host.

    10. Again select Approve host.

    11. Notice sno1.lab.example.com host is in Ready status.

      Once sno1.lab.example.com host is in Ready status, click Next

  10. In networking section, you may notice the Some validations failed message.

    After few minutes, the message goes away and notice the host is in Ready status.

  11. If you notice All checks passed for cluster and host validations then click Install cluster.

  12. Notice the installation has started.

  13. After 7 to 10 minutes, it waits on pending user action.

    This means you need to disconnect the discovery ISO from the sno1 VM and boot the sno1 VM from disk.

    1. Shutdown the sno1 VM.

    2. Update the boot order to boot sno1 VM from disk.

  14. After 2 minutes, installation proceeds and you will notice the progress.

  15. Installation completes in approximately in 20 minutes.

  16. Notice the sno1 is added to cluster list in default cluster set.

    This concludes successful deployment of OpenShift cluster and added to hub cluster using RHACM.

Install remaining Infrastructure clusters as SNO clusters

  1. You can deploy remaining sno2 and sno3 clusters by following steps from previous section followed for sno1 cluster deployment.

  2. Each cluster deployment take 35 to 40 minutes to complete.
