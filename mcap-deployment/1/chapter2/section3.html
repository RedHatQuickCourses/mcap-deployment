Access the Hub Cluster

In this section, you will be accessing Hub cluster.

Prerequisites

  1. Ensure Hub cluster is deployed successfully.

  2. Ensure /root/hub/ directory and file structure created.

    Sample output:
    [root@hypervisor ~]# ls -al /root/hub/
total 28
drwxr-xr-x.  2 root root  4096 Aug 22 15:20 .
dr-xr-x---. 13 root root  4096 Aug 22 15:18 ..
-rw-r--r--.  1 root root    94 Aug 22 15:11 console-url
-rw-r--r--.  1 root root    24 Aug 22 15:11 kubeadmin-password
-rw-r--r--.  1 root root 12127 Aug 22 15:20 kubeconfig

  3. Get the kubeconfig file, the password for kubeadmin user, and the web console URL from console.redhat.com.

    1. Download the kubeconfig file to your laptop or desktop and then copy to /root/hub directory on hypervisor.

    2. Copy the password for kubeadmin user and paste it in /root/hub/kubeadmin-password file.

    3. Copy the web console url and paste it in /root/hub/console-url file.

  4. Ensure /root/.kube directory and config file in /root/.kube directory.

    Sample output:
    [root@hypervisor ~]# ls -al /root/.kube
total 36
drwxr-xr-x.  3 root root  4096 Sep  4 13:19 .
dr-xr-x---. 18 root root  4096 Sep  5 08:28 ..
-rw-r--r--.  1 root root 20513 Sep  5 08:29 config

Access the Hub Cluster via CLI

  1. Download the OpenShift client (oc).

    wget https://mirror.openshift.com/pub/openshift-v4/clients/ocp/latest/openshift-client-linux.tar.gz

  2. Extract the archive.

    tar -xzvf openshift-client-linux.tar.gz

  3. Move oc and kubectl binaries to /usr/local/bin directory.

    mv oc /usr/local/bin/; mv kubectl /usr/local/bin/

  4. Copy /root/hub/kubeconfig file as /root/.kube/config file.

    cp /root/hub/kubeconfig /root/.kube/config

  5. Set kubepass variable as kubeadmin user’s password.

    kubepass=$(cat /root/hub/kubeadmin-password)

  6. Login to Hub cluster with oc login command.

    oc login -u kubeadmin -p $kubepass
    Sample output:
    Login successful.

You have access to 79 projects, the list has been suppressed. You can list all projects with 'oc projects'

Using project "default".

  7. Review the nodes.

    oc get nodes
    Sample output:
    NAME                  STATUS   ROLES                         AGE   VERSION
hub.lab.example.com   Ready    control-plane,master,worker   10h   v1.29.7+6abe8a1

Access the Hub Cluster from Web Console

  1. Access the Firefox browser on console of hypervisor using VNC viewer.

  2. Get the web console url from /root/hub/console-url file.

    Select the url and paste in firefox browser tab.

    Click Advanced…​ to proceed.

    Click Accept the Risk and Continue to proceed.

    You may need to accept the risk twice.

  3. Login as kubadmin user.

    Get the kubadmin user’s passwrod from /root/hub/kubeadmin-password file.

  4. Once you logged in as kubadmin user, this is what the first screen looks like.

  5. Verify your local-cluster i.e Hub cluster is in Ready state.
