Install and Configure Operators

In this section, you will be installing additional operators needed for deploying Tenant cluster.

Prerequisites

  1. Verify that sno1 cluster is deployed successfully.

  2. Access the sno1 cluster via CLI and web console.

  3. Ensure that sno1.lab.example.com node is in Ready status and all cluster operators are available.

    Sample output:
    [root@hypervisor ~]# oc get nodes

NAME                   STATUS   ROLES                         AGE   VERSION
sno1.lab.example.com   Ready    control-plane,master,worker   23h   v1.29.7+4510e9c

[root@hypervisor ~]# oc get clusterversion

NAME      VERSION   AVAILABLE   PROGRESSING   SINCE   STATUS
version   4.16.8    True        False         23h     Cluster version is 4.16.8

Install OpenShift Virtualization Operator

  1. Access the operator hub from the web console of sno1 cluster.

    From the left navigation pane, click Operators  OperatorHub.

  2. In a search window, search openshift virtualization and select the OpenShift Virtualization.

  3. Click Install to open the install options.

  4. Keep all options as is, with no change in selected options and then click Install to install the operator.

  5. Click Create HyperConverged to create the resource.

  6. Keep all options as is, with no change, in selected options and then click Create to create the resource.

  7. Notice it goes into Progressing, Degraded conditions.

  8. After 3-4 minutes, notice the Refresh web console message on window.

  9. After 3-4 minutes, HyperConverged is in Reconcile, Complete, Available, Upgradable conditions.

Install OpenShift Data Foundation Operator

  1. Access the operator hub from the web console of sno1 cluster.

    From the left navigation pane, click Operators  OperatorHub.

  2. In a search window, search odf and select the OpenShift Data Foundation.

  3. Click Install to open install options.

  4. Keep all options as is, with no change, in selected options and then click Install to install the operator.

  5. After 3-4 minutes, notice the Refresh web console message on window.

    First, refresh the web console and then click Create StorageSystem to create the resource.

  6. Create the StorageSystem.

    1. Provide backing storage details.

      Deployment type: Full deployment.

      Select Connect an external storage platform.

      Storage platform: Red Hat Ceph Storage.

      Check box the Use Ceph RBD as the default StorageClass.

      Click Next to proceed.

    2. Provide connection details.

      Use the Download script option to download the Python script.

      You will need to run this script on the storage VM.

      scp /root/Downloads/ceph-external-cluster-details-exporter.py root@storage:.

      Password for root user is redhat.

      Sample output:
      [root@hypervisor ~]# ls /root/Downloads/
ceph-external-cluster-details-exporter.py

[root@hypervisor ~]# scp /root/Downloads/ceph-external-cluster-details-exporter.py root@storage:.
root@storage's password:
ceph-external-cluster-details-exporter.py

      Run the ceph-external-cluster-details-exporter.py script on the storage VM and store output in output.json file.

      python ceph-external-cluster-details-exporter.py --rbd-data-pool-name default.rgw.control > output.json
      Sample output:
      [root@hypervisor ~]# ssh root@storage
root@storage's password:

[root@storage ~]# ceph osd lspools
1 .rgw.root
2 .mgr
3 default.rgw.log
4 default.rgw.control
5 default.rgw.meta

[root@storage ~]# python ceph-external-cluster-details-exporter.py --rbd-data-pool-name default.rgw.control > output.json

      Copy the output.json file from storage VM to hypervisor.

      Run following command on hypervisor.

      scp root@storage:/root/output.json /root/
      Sample output:
      [root@hypervisor ~]# scp root@storage:/root/output.json .
root@storage's password:

Install NMState Operator

  1. Access the operator hub from the web console of sno1 cluster.

    From the left navigation pane, click Operators  OperatorHub.

  2. In a search window, search NMstate and select the Kubernetes NMState Operator.

  3. Click Install to open install options.

  4. Keep all options as is, with no change, in selected options and click Install to install the operator.

  5. Click View Operator to view the operator details.

  6. In NMState tab and click Create NMState to create the resource.

  7. Keep all options as is, with no change, in selected options and then click Create to create the resource.

  8. Notice nmstate resource is created.

  9. After 3-4 minutes, notice the Refresh web console message on window.
