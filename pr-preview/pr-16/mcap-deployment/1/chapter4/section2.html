Assisted Clusters - Tenant Cluster

In this section, you will be deploying the Tenant cluster using RHACM from the Hub cluster. The Tenant cluster will be the Three-Node OpenShift Compact cluster.

Prerequisites

The tcn1.lab.example.com, tcn2.lab.example.com, and tcn3.lab.example.com VMs (created using OpenShift Virtualization) are up and running.

Deploy Tenant cluster as Three-Node OpenShift Compact Cluster

  1. Login to the web console of the Hub cluster.

    Ensure that you have switched to All Clusters from the local-cluster.

  2. Create the cluster using Create cluster.

  3. Select Host inventory.

  4. Select Standalone.

  5. Select Add new hosts.

  6. Provide cluster details.

    Cluster name: tenant

    Cluster set: default

    Base domain: lab.example.com

    Select 4.16.8 version of OpenShift from the menu.

    1. Get the pull secret from pull_secret.txt and provide the pull secret in the Pull secret field.

      Click Next

  7. No automation template.

    Click Next

  8. Review and save.

    Click Save

  9. Add tcn1.lab.example.com, tcn2.lab.example.com, and tcn3.lab.example.com VMs as host.

    1. Select Add hosts  With Discovery ISO.

    2. Here you will need to provide the public SSH key of the root user.

    3. Get the public SSH key of the root user.

    4. Provide the public SSH key of the root user in the SSH public key field.

      Click Generate Discovery ISO to generate discovery ISO.

    5. Click Download Discovery ISO to download discovery ISO on the hypervisor.

    6. Upload ISO from /root/Download directory to Infrastructure clusters.

      Login to the sno1 cluster.

      Sample output:

      Upload the discovery ISO using the virtctl image-upload command.

      Sample output:
      [root@hypervisor ~]# ls /root/Downloads/

      Verify that the PVC is created on the sno1 cluster.

      In the sno1 cluster web console, from the left navigation pane; click Storage  PersistentVolumeClaims.

      Upload the discovery ISO to sno2 and sno3 clusters by performing the above steps.

    7. Boot the tcn1.lab.example.com, tcn2.lab.example.com, and tcn3.lab.example.com VMs with discovery ISO.

      In the sno1 cluster web console, from the left navigation pane; click Virtualization  VirtualMachines.

      Stop the tcn1.lab.example.com VM.

      Verify that the tcn1.lab.example.com VM is stopped.

      From the tcn1.lab.example.com VM’s Configuration tab, select Storage.

      Click Add disk to add the discovery ISO as the PVC.

      Select Source  PVC and then select Select PersistentVolumeClaim  tenant-iso-pvc.

      Keep the interface as VirtIO and click Save to add the disk.

      Edit the boot order of the tcn1.lab.example.com VM from the Configuration tab, and select Details.

      Move up the newly added disk at the top and click Save.

      Start the tcn1.lab.example.com VM.

      Ensure the tcn1.lab.example.com VM boots with the discovery ISO.

      Follow the same steps above for the tcn2.lab.example.com and tcn3.lab.example.com VMs to boot them with the discovery ISO.

    8. Return to the web console of the Hub cluster to proceed with the cluster installation.

      Approve the discovered host tcn1.lab.example.com.

      Ensure that the discovered host tcn1.lab.example.com is ready.

      Approve the remaining hosts tcn2.lab.example.com and tcn3.lab.example.com.

      Click Next to proceed.

  10. In the networking section, ensure all hosts are ready.

    Provide the API IP and Ingress IP from the zone file.

    Click Next to proceed.

  11. If you notice All checks passed for the cluster and host validations, then click Install cluster.

  12. Notice that the installation has started.

  13. After 7 to 10 minutes, the installation will wait for Pending user action.

    This means that you will need to disconnect the discovery ISO from the tcn3.lab.example.com VM and boot the tcn3.lab.example.com VM from disk.

    This means that you will also need to disconnect the discovery ISO from the tcn2.lab.example.com VM, and boot the tcn2.lab.example.com VM from disk.

    1. Shutdown the tcn2.lab.example.com VM.

    2. Update the boot order to boot the tcn2.lab.example.com VM from disk.

      Ensure that the tcn2.lab.example.com VM boots from disk.

      Follow the same steps above to boot the tcn3.lab.example.com VM from disk.

  14. After 2 minutes, installation proceeds and you will notice the progress.

    After 5 minutes, the tcn2.lab.example.com and tcn3.lab.example.com nodes are installed.

  15. Installation proceeds and continues with tcn1.lab.example.com node.

  16. After 7 to 10 minutes, the installation will wait for Pending user action.

    This means that you will need to disconnect the discovery ISO from the tcn1.lab.example.com VM and boot the tcn1.lab.example.com VM from disk.

    Follow the same steps that you followed for tcn2.lab.example.com VM to boot the tcn1.lab.example.com VM from disk.

  17. Installation will complete in approximately 20 minutes.

  18. Notice that the tenant cluster is added to the cluster list in the default cluster set.

    This concludes the successful deployment of the OpenShift cluster and added to hub cluster using RHACM.
