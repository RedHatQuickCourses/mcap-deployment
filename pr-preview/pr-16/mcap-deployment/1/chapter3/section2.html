Assisted Clusters - Infrastructure Clusters

In this section, you will be deploying Infrastructure clusters using RHACM from the Hub cluster. These Infrastructure clusters will be Single Node OpenShift (SNO) clusters.

Prerequisites

  1. sno1, sno2, and sno3 VMs are up and running.

  2. Download or copy your pull-secret and put it in pull_secret.txt file on the hypervisor.

  3. Set the OpenShift clusterimageset to 4.16.8 on a stable channel.

    This will allow you to deploy the Infrastructure and Tenant clusters with the same OpenShift version.

    1. Get the images with visible label as true from clusterimageset.

      Sample output:
      [root@hypervisor ~]# oc get clusterimageset -o yaml | grep -i true -A 1

      visible: "true"
    name: img4.14.35-multi-appsub
--
      visible: "true"
    name: img4.15.29-multi-appsub
--
      visible: "true"
    name: img4.16.10-multi-appsub

    2. Set the visible label as true and channel label as stable for 4.16.8 image.

      oc patch clusterimagesets.hive.openshift.io img4.16.8-multi-appsub --type='json' -p='[{"op":"replace","path": "/metadata/labels/visible","value":"true"}]'
      oc patch clusterimagesets.hive.openshift.io img4.16.8-multi-appsub --type='json' -p='[{"op":"replace","path": "/metadata/labels/channel","value":"stable"}]'
      Sample output:
      [root@hypervisor ~]# oc patch clusterimagesets.hive.openshift.io img4.16.8-multi-appsub --type='json' -p='[{"op":"replace","path": "/metadata/labels/visible","value":"true"}]'
clusterimageset.hive.openshift.io/img4.16.8-multi-appsub patched

Deploy Infrastructure clusters as SNO clusters

  1. Login to the web console of the Hub cluster.

    Switch to the All Clusters from the local-cluster.

  2. Create a cluster using Create cluster.

  3. Select Host inventory.

  4. Select Standalone.

  5. Select Add new hosts.

  6. Provide cluster details.

    Cluster name: sno1

    Cluster set: default

    Base domain: lab.example.com

    Select 4.16.8 version of OpenShift from the menu.

    Check the box Install single node OpenShift (SNO).

    1. Get the pull secret from pull_secret.txt.

    2. Provide the pull secret in the Pull secret field.

      Click Next

  7. No automation template.

    Click Next

  8. Review and save.

    Click Save

  9. Add sno1 VM as host.

    Click Add hosts.

    1. Select Add hosts  With Discovery ISO.

    2. Here you need to provide the public ssh key of the root user.

    3. Get the public ssh key of the root user.

    4. Provide the public ssh key of the root user in the SSH public key field.

      Click Generate Discovery ISO to generate discovery ISO.

    5. Click Download Discovery ISO to download discovery ISO on the hypervisor.

    6. This will open the link in the new tab.

      Click Advanced…​ to proceed.

      Click Accept the Risk and Continue to proceed.

    7. Move ISO from Download directory to /var/lib/libvirt/images/ directory.

      Sample output:
      [root@hypervisor ~]# mv Downloads/39c2b646-e122-435b-b85e-c8ec6dd18d04-discovery.iso /var/lib/libvirt/images/

    8. Boot the sno1 VM with discovery ISO.

      Shutdown the sno1 VM.

      Click Add hardware

      Select Device type  CDROM device and select Select or create custom image option.

      Click Manage.

      Select the discovery ISO and click Choose Volume.

      Select the discovery ISO and click Finish.

      Update the Boot device order to boot the system with discovery ISO.

      Click Apply.

      Boot the sno1 VM and ensure it is booted with RHEL CoreOS (Live).

    9. In the hub console, notice sno1 VM as host is discovered and select Approve host.

    10. Again select Approve host.

    11. Notice sno1.lab.example.com host is in Ready status.

      Once sno1.lab.example.com host is in Ready status, click Next

  10. In the networking section, you may notice the Some validations failed message.

    After a few minutes, the message goes away and notice that the host is in the Ready status.

  11. If you notice All checks passed for cluster and host validations then click Install cluster.

  12. Notice the installation has started.

  13. After 7-10 minutes, it waits for pending user action.

    This means you need to disconnect the discovery ISO from the sno1 VM and boot the sno1 VM from the disk.

    1. Shutdown the sno1 VM.

    2. Update the boot order to boot sno1 VM from the disk.

  14. After two minutes, installation proceeds and you will notice the progress.

  15. Installation completes in approximately 20 minutes.

  16. If you notice any failure in importing the cluster to the Hub cluster then wait for 35 to 40 minutes.

    1. Detach the cluster.

      Click Actions  Detach cluster

    2. Try to import manually again.

      Click Actions  Import cluster

      Click Import to import cluster.

  17. Notice that sno1 is added to the cluster list in the default cluster set.

    This concludes the successful deployment of the OpenShift cluster and added to the hub cluster using RHACM.

Install remaining Infrastructure clusters as SNO Clusters

  1. You can deploy the remaining sno2 and sno3 clusters by following the steps from the previous section for sno1 cluster deployment.

  2. Each cluster deployment will take 35 to 40 minutes to complete.
