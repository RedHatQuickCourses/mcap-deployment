Access the Infrastructure Cluster

In this section, you will be accessing Infrastructure clusters.

Prerequisites

  1. Ensure that Infrastructure clusters (sno1, sno2 and sno3) are deployed successfully.

  2. Ensure /root/sno1/ directory and file structure created.

    Sample output:
    [root@hypervisor ~]# ls -al /root/sno1/
total 28
drwxr-xr-x.  2 root root  4096 Aug 22 15:20 .
dr-xr-x---. 13 root root  4096 Aug 22 15:18 ..
-rw-r--r--.  1 root root    24 Aug 22 15:11 kubeadmin-password
-rw-r--r--.  1 root root 12127 Aug 22 15:20 kubeconfig

  3. Get the kubeconfig file and password for kubeadmin user from the Hub cluster console.

    1. Download the kubeconfig file to hypervisor, and then copy to /root/sno1 directory on hypervisor.

      Sample output:
      [root@hypervisor ~]# mv /root/Downloads/sno1-kubeconfig.yaml /root/sno1/kubeconfig

    2. Copy the password for kubeadmin user, and paste it in new tab of a Firefox browser.

      Copy the password from the tab of Firefox browser, and paste it in /root/sno1/kubeadmin-password file.

Follow the same steps for sno2 and sno3 clusters.

Access the sno1 Cluster via CLI

  1. Copy the /root/sno1/kubeconfig file as /root/.kube/config file.

    cp /root/sno1/kubeconfig /root/.kube/config

  2. Set the kubepass variable as kubeadmin user’s password.

    kubepass=$(cat /root/sno1/kubeadmin-password)

  3. Login to sno1 cluster with oc login command.

    oc login -u kubeadmin -p $kubepass
    Sample output:
    Login successful.

You have access to 79 projects, the list has been suppressed. You can list all projects with 'oc projects'

Using project "default".

  4. Review the nodes.

    oc get nodes
    Sample output:
    NAME                  STATUS   ROLES                         AGE   VERSION
sno1.lab.example.com   Ready    control-plane,master,worker   10h   v1.29.7+6abe8a1
Follow the same steps for sno2 and sno3 clusters.

Access the sno1 Cluster from Web Console

  1. Get the web console url from Hub cluster console.

  2. Click on the link from Web Console URL.

    Click Advanced…​ to proceed.

    Click Accept the Risk and Continue to proceed.

    You may need to accept the risk twice.

  3. Login as kubadmin user.

    Get the kubadmin user’s password from from Hub cluster console.

    Copy the kubadmin user’s password from from Hub cluster console and paste it in Password field.

  5. Verify sno1 cluster is in Ready state in Hub cluster console.

Access the sno2 and sno3 Clusters via CLI and from Web Console

  1. Follow the same prerequisites from the previous section for sno2 and sno3 clusters.

  2. Follow same the steps from the previous section for accessing the sno2 and sno3 clusters.
